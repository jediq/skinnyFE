SkinnyFE

Say goodbye to bloated middleware.

SkinnyFE lets you render RESTful back-end data in HTML with minimal coding.

SkinnyFE is a Front End framework that removes the ability to embed business logic in a front end application.

At it's core, SkinnyFE declares within the HTML template the RESTful API resources that are required to render the page. These resource references are then looked up in configuration and HTTP calls are made. Finally, the response data from the resources is made available to the HTML template to embed in the output HTML.

SkinnyFE renders on the server side. That doesn't mean you can only use server side rendering, the HTML that it renders can obviously contain javascript, it just means that you don't need it. Rendering on the server first and then progressively enhancing with javascript later means that you can still treat clients with #nojs requirements (such as screen-readers) as first class citizens.

So, what does it look like?

We aim to keep the configuration as light touch as possible, choosing convention over configuration where we can. SkinnyFE requires a simple Json config file and at least one moustache template (roots of folders are resolved to index.moustache).

config.json 

{
    "port":8027,
    "resources":[
        {
            "name":"Posts",
            "url":"http://localhost:8009/posts"
        }
    ]
}
index.moustache 

<!DOCTYPE html>
<html lang="en-US">

    <head>
    <meta property="posts" resource="Posts"/>
    </head>
    <body>
        <h1>Posts</h1>
        {{#each posts}}
            <h2>{{title}} by {{author}}</h2>
        {{/each}}
    </body>
</html>

That's all you need, the SkinnyFE JAR file distributable contains all of the libraries required, including an embedded web server, so launching the application is simple a case of telling it where the config is: 

$ java -jar SkinnyFE-all.jar config.json
